# Compiling Reality

A structural framework — the **Algebra of Four-Fold Distinction** — developed bottom-up from two questions asked persistently for several decades:

**How does this work?**
**What is the minimum requirement for X?**

The answer that kept returning: four variables in complete mutual determination, twelve equations, one tetrahedral volume. The same structure, recurring across domains that do not talk to each other.

## The Architecture of the Files

This repository refuses a single, centralized "Theory of Everything" document because doing so would commit the exact error the framework diagnoses: collapsing a multi-dimensional volume into a flat surface. 

Instead, the corpus is distributed across different epistemic registers. The file prefixes tell you exactly what level of detail, what kind of proof, and what standard of rigor you are about to encounter:

*   **`L0` – `L5` (The Core Progression):** The formal spine. These documents trace the structure from its foundational orientation (`L0`), through the 12 static equations of mutual determination (`L1`-`L3`), up to the dynamic AC extension, dimensional collapse, and deployed swarms (`L4`-`L5`).

*   **`Proof*` (The Proof Suite):** Domain applications that fall out of the framework rather than being imposed on it. These are not all "mathematical" proofs; they include structural proofs and proofs by load-bearing paradox. Every proof states its Claim, Type, Method, and Falsification Condition at the very top.

*   **`QED-*` (The Hard Stops):** The sharpest edges of the corpus. These contain hard, falsifiable predictions (e.g., `QED-CCP` and the Casimir null-gravity signature) or raw demonstrations of the framework in live action (`QED-Recursion`).

*   **`Supplement-*` (The Worked Examples & Ledgers):** The messy, necessary work of mapping the structure to reality. These files contain detailed derivations, historical groundings, worked diagnostic examples, etc.

*   **`Meta-*` (The Boundary Conditions):** The framework turned on itself. These documents examine the transitions, the nature of decay, and the ultimate Tangent Singularity, ensuring the system does not accidentally claim to have conquered the infinite.

*   **`TooTrue-*` (The Mythopoetic Register):** Phenomenological fiction. These documents use narrative to demonstrate what topological distortions actually *feel* like from the inside, deliberately breaking formal rules to map the inner weather of the system.

## The Executable Topology

This repository is not just a text. It contains `code/`, `code/manifolds/`, and `test/`.

In standard theoretical frameworks, claims remain on the page as metaphors. Here, the structure is mechanized. The directories contain Prolog constraint engines, Lisp evaluators, Python AST checkers, and the deployed `AlgebraicSwarmController`. 

*   The **`test/`** directory parses the abstract syntax trees of the equation set to mechanically prove that the arithmetic operators enforce the framework's topological laws (the Numerator Law, the Route Law).
*   The **`code/manifolds/`** directory houses the Tribunal Manifold—a diagnostic engine that takes human or systemic state inputs and computes the structural drift, outputting the specific K4 pathology without human intervention.
*   The **`code/`** directory proves that the 12-step Braid and the equations of mutual determination can stably orchestrate synthetic agents in the wild.

The code is the ultimate falsifier. It proves that the framework is not a subjective mapping heuristic, but an auto-classifying arithmetic syntax. If you feed it a topological impossibility, the compiler crashes.
